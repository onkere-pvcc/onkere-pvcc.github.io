Protocols CSMA/CD Ethernet Carrier-sense multiple access with collision detection (CSMA/CD) is a medium access control (MAC) method used most notably in early Ethernet technology for local area networking. It uses carrier-sensing to defer transmissions until no other stations are transmitting.
CSMA/CA WiFi Wi-Fi devices share the same medium, RF. In order to avoid collisions with other devices connected to the same AP and those nearby, all devices use a form of contention called Carrier Sense Multiple Access with Collision Avoidance (CSMA/CA)
Repeater A repeater is a device that retransmits a signal with more power or amplifies an input signal for retransmission1,2,3,4. It is used in computer networks to expand the coverage area, service remote nodes, or overcome the distance limit of a single span of network cable
Bridge The bridge is a networking device in a computer network that is used to connect multiple
Switch A switch in networking is a device that connects other devices on a local area network (LAN) and manages the flow of data by sending incoming packets to their intended destination based on their address
Router A router is a networking device that connects two or more networks or subnetworks and forwards data packets between them1,2,3. It allows multiple devices to share a single Internet connection2,4 and manages traffic by sending data to their intended IP addresses2,3. A router can link computers, printers, and other devices in your home network to the Internet
Gateway A gateway is a piece of networking hardware or software used in telecommunications networks that allows data to flow from one discrete network to another
Client/server model The clientâ€“server model is a distributed application structure that partitions tasks or workloads between the providers of a resource or service, called servers, and service requesters, called clients
P2P: peer-to-peer Peer-to-peer (P2P) computing or networking is a distributed application architecture that partitions tasks or workloads between peers
distributed system A distributed system is any network structure that consists of autonomous computers that are connected using a distribution middleware. Distributed systems facilitate sharing different resources and capabilities, to provide users with a single and integrated coherent network
cloud computing Cloud computing is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g., networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction.
Internet Addressing Tier 1 A Tier 1 network is an Internet Protocol (IP) network that can reach every other network on the Internet solely via settlement-free interconnection (also known as settlement-free peering). Tier 1 networks can exchange traffic with other Tier 1 networks without paying any fees for the exchange of traffic in either direction
Tier 2 A Tier 2 network is an Internet service provider which engages in the practice of peering with other networks, but which also purchases IP transit to reach some portion of the Internet.
Tier 3 A data center with multiple paths for power and cooling, and redundant systems that allow the staff to work on the setup without taking it offline. This tier has an expected uptime of 99.982% per year.
IP address ICANN Stands for "Internet Corporation For Assigned Names and Numbers." The ICANN is an non-profit corporation that is responsible for allocating IP addresses and managing the domain name system.
Multi-factor authentication Multi-factor Authentication (MFA) is an authentication method that requires the user to provide two or more verification factors to gain access to a resource such as an application, online account, or a VPN. MFA is a core component of a strong identity and access management (IAM) policy.
